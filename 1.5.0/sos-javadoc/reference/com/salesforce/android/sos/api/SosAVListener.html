SosAVListener

com.salesforce.android.sos.api.SosAVListener

Class Overview

Interface to listen for changes in the sessions audio or video status.

You can use this interface to be alerted whenever the agent or user mutes the video, or whenever the user mutes his microphone.


Example Usage

 
   SosAVListener myListener = new SosAVListener() {
     @Override
     public void onLocalAudioToggled (boolean enabled) {
       // Called whenever the user toggles the mute button.
       // `enabled` conveys the updated state.
     }

     @Override
     public void onVideoToggled (boolean enabled) {
       // Called whenever the user or agent mutes the agent video.
       // `enabled` conveys the updated state.
     }
   };

   Sos.addAVListener(myListener);

   // Sometime later ...
   // No longer be alerted about audio or video events
   Sos.removeAVListener(myListener);

public abstract void onVideoToggled (boolean enabled)

Called whenever the user or agent enables/disables the agent video feed during a call.

Note that since video is synchronized between agent and customer, the state is reported once the synchronization is complete. Because of this, the event may not fire immediately. It will fire after one round trip to the agent.

Parameters
enabled represents the new video state
